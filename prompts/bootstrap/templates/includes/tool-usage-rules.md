## Правила работы с инструментами

### ПРАВИЛО 1 — Проверяй параметры
Перед КАЖДЫМ вызовом инструмента убедись, что ВСЕ обязательные параметры указаны с точными именами.
Если хоть один параметр пропущен или имя неверное — исправь ДО вызова.

### ПРАВИЛО 2 — Абсолютные пути
`file_path` ВСЕГДА абсолютный (начинается с `/`). Никогда не передавай относительный путь.
Если путь неизвестен — сначала найди файл через `glob` или `grep_search`.

### ПРАВИЛО 3 — Самовосстановление
Если инструмент вернул ошибку (missing param, invalid value, etc.):
1. Прочитай сообщение об ошибке
2. Определи какой параметр неверный / пропущенный
3. Исправь параметры
4. Повтори вызов
НЕ пропускай шаг и НЕ переходи к следующему действию без результата инструмента.

### ПРАВИЛО 4 — Не угадывай
Если значение параметра неизвестно — получи его другим инструментом:
- Не знаешь путь к файлу? → `glob` или `grep_search` → затем `read_file`
- Не знаешь содержимое? → `read_file` → затем `edit`
- Нужно убедиться что файл существует? → `glob` → если найден → `read_file`

### ПРАВИЛО 5 — Валидируй перед edit
Перед вызовом `edit`:
1. Прочитай файл через `read_file`
2. Убедись что `old_string` ТОЧНО совпадает с содержимым (включая отступы, переносы)
3. Если `old_string` не найден — прочитай файл заново и попробуй снова

### Обязательные параметры инструментов

| Инструмент | Обязательные параметры | Частые ошибки |
|-----------|----------------------|---------------|
| `read_file` | `file_path` (абсолютный путь) | Относительный путь, опечатка в `file_path` |
| `edit` | `file_path`, `old_string`, `new_string` | Неверный `old_string`, `file_path` не абсолютный |
| `write_file` | `file_path`, `content` | Относительный путь |
| `run_shell_command` | `command` | Пустая команда, команды с интерактивным вводом |
| `grep_search` | `pattern` | Специальные regex-символы без экранирования |
| `glob` | `pattern` | Пустой паттерн |
| `list_directory` | `path` (абсолютный) | Относительный путь |
| `ask_user_question` | `questions` (array) | Пустой массив questions |
| `save_memory` | `fact` | Длинный текст вместо concise fact |

### Пример корректного вызова

**НЕВЕРНО** (пропущен file_path):
```
read_file()
```

**НЕВЕРНО** (относительный путь):
```
read_file(file_path="app/Models/User.php")
```

**ВЕРНО**:
```
read_file(file_path="/mnt/f/Projects/php/courts/courts/app/Models/User.php")
```
